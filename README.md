# ГофроКалькулятор FEFCO

Веб-приложение для отдела продаж завода гофротары: расчёт стоимости гофрокоробки с
коэффициентами наценки по объёму заказа, надбавкой по площади заготовки, схемой
коробки, экспортом коммерческого предложения в PDF и технологической карты в Excel.

Работает на телефоне и компьютере, не требует сервера и аутентификации, хранит
настройки в браузере. Сам расчёт и схема работают офлайн; интернет нужен только при
экспорте (с CDN подгружаются jsPDF, jsPDF-AutoTable и ExcelJS).

---

## Что внутри

| Файл | Назначение |
| --- | --- |
| `index.html` | Готовое приложение (в репозитории — собранная single-file версия для GitHub Pages, открывается сразу). |
| `competitors.json` | Данные цен конкурентов (см. раздел «Цены конкурентов» ниже). Редактируется прямо на GitHub. |

> **Важно про competitors.json:** это **отдельный** файл рядом с `index.html`. Приложение читает
> его через `fetch('./competitors.json')` при загрузке страницы, поэтому файл **не встраивается**
> в HTML и должен лежать в том же каталоге, что и `index.html` (на GitHub Pages это выполняется
> автоматически). При локальном открытии через `file://` браузер блокирует такой запрос — блок
> цен конкурентов просто не показывается, остальной расчёт работает нормально.

### Исходники (для разработчика)

Разделённая версия и сборщик хранятся в рабочем каталоге проекта (не обязательны для работы сайта):
`index.html` + `app.js` + `fonts.js` — исходники; `build.py` инлайнит `fonts.js` и `app.js` в
`gofrokalkulyator.html` и копирует результат (вместе с `competitors.json`) в `deploy/` и
`github-pages/`.

## Возможности

- Ввод длины (A), ширины (B), высоты (H) в мм с валидацией минимальных размеров
  (длина ≥ 245, ширина ≥ 80, ширина + высота ≥ 320 — настраивается).
- **Категории коробок** — «Четырёхклапанные» и «Самосборные», у каждой свой набор
  коэффициентов наценки по градациям.
- **Типы развёртки (11 типов FEFCO)** — привязаны к категории. Площадь заготовки для любого
  типа: **S = f1 × f2 / 1 000 000** (м²), где f1 — ширина заготовки (мм), f2 — высота
  заготовки (мм); A — длина (Д), B — ширина (Ш), H — высота (В).

  **Четырёхклапанные (набор коэффициентов А):**

  | FEFCO | f1 (ширина заг.) | f2 (высота заг.) |
  | --- | --- | --- |
  | 0201 | 2Д + 2Ш + 60 | Ш + В + 6 |
  | 0203 | 2Д + 2Ш + 60 | 2Ш + В + 6 |
  | 0215 | 2Д + 2Ш + 60 | 2Ш + В + 30 |
  | 0216 | 2Д + 2Ш + 80 | 2Ш + В + 26 |
  | 0217 | 2Д + 2Ш + 80 | 2Ш + В + 26 |

  **Самосборные (набор коэффициентов Б):**

  | FEFCO | f1 (ширина заг.) | f2 (высота заг.) |
  | --- | --- | --- |
  | 0409 | Д + 2В + 6 | 2Ш + 3В + 12 |
  | 0422 | Ш + 2В + 66 | Д + 4В + 66 |
  | 0426 | Д + 2В + 30 | 2Ш + 4В + 50 |
  | 0427 | Д + 4В + 70 | 2Ш + 3В + 40 |
  | 0443 | 2Ш + 2В + 67 | Д + 2В + 27 |
  | 0469 | Д + 4В + 120 | 2Ш + 2В + 66 |

  Плюс **пользовательские типы** с собственными формулами длины/ширины заготовки (задаётся
  категория, к которой относится тип).
- **Проверка размеров** — ограничения (длина ≥ 245, ширина ≥ 80, ширина + высота ≥ 320)
  применяются **только к четырёхклапанным** и **не блокируют** расчёт: при нарушении
  показывается оранжевое предупреждение, но расчёт, PDF и техкарта остаются доступны.
  Самосборные типы от этой проверки освобождены.
- **Цены конкурентов** — блок рядом с ценой без НДС, данные из `competitors.json`
  (см. отдельный раздел ниже).
- **Коэффициенты наценки** по градациям объёма (50–99, 100–299, 300–499, 500–999, 1000+),
  редактируются на вкладке «Настройки» отдельно для каждой категории.
- **Надбавка к коэффициенту по площади заготовки** — до трёх правил, каждое с признаком
  активности, границами «от»/«до» (м²) и величиной надбавки. Применяется первое
  подходящее активное и заполненное правило (сверху вниз). См. раздел ниже.
- **Ручной режим количества**: при вводе точного количества вручную автоподбор градации
  отключается — выбранная градация и коэффициент фиксируются. Ссылкой «Включить автоподбор»
  можно вернуть автоматический выбор по количеству.
- Скидка клиента в % (по умолчанию 0).
- Схема коробки в изометрии с корректными пропорциями: длина по нижнему ребру дна,
  подписи сторон соотнесены со стенками.
- Внутренний объём (л), вес одной коробки и партии (кг), габаритный объём партии (м³).
- Справочная **маржа** (валовая и чистая с учётом накладных) — только в интерфейсе и Excel,
  в коммерческое предложение не попадает.
- **Экспорт PDF** — одностраничный бланк коммерческого предложения с логотипом (кириллица,
  встроенный PT Sans). Без коэффициентов, надбавок, градаций и маржи — готов к отправке клиенту.
- **Экспорт Excel** — технологическая карта: раскрой заготовки (для FEFCO 0201 — полная
  техкарта с беговками, ножами и строкой **«Корректировка»** = (H + клапан − 430) × 1.187;
  для остальных типов — размеры и площадь заготовки), коммерческий расчёт с разбивкой
  коэффициента и справочная маржа. Все производные значения — формулы Excel, доступные для аудита.

## Формула расчёта цены

Актуальная логика: базовая цена задаётся **с НДС**, цена без НДС вычисляется автоматически.

```
Итоговый коэффициент = Базовый коэффициент (по тиражу) + Надбавка по площади

Цена 1 шт (с НДС)  = Площадь заготовки (м²) × Итоговый коэффициент × (1 − Скидка)
Цена 1 шт (без НДС) = Цена с НДС / (1 + НДС/100)
```

Ставка НДС (по умолчанию 20 %), цена картона за м² и плотность (для расчёта маржи и веса),
припуски и ограничения размеров задаются в «Настройках».

## Надбавка к коэффициенту по площади заготовки

Блок в «Настройках» из трёх строк. Каждая строка: признак активности, «Площадь от, м²»,
«Площадь до, м²», «Прибавить к коэффициенту».

Логика применения (первое подходящее активное и заполненное правило, сверху вниз):

| Заполнено | Правило срабатывает при |
| --- | --- |
| только «до» | площадь заготовки ≤ значения |
| только «от» | площадь заготовки ≥ значения |
| оба поля | площадь в диапазоне «от–до» включительно |

Значение надбавки прибавляется к базовому коэффициенту по тиражу. В расчёте показываются
базовый коэффициент, надбавка и итоговый коэффициент.

## Цены конкурентов (competitors.json)

Блок «Цены конкурентов» появляется рядом с ценой без НДС, когда для текущего расчёта
находится подходящая запись в файле `competitors.json`.

**Как это работает:**

- При загрузке страницы приложение один раз читает `competitors.json` (рядом с `index.html`).
- Для текущего расчёта ищется **точное совпадение** по коду FEFCO, размерам (Д/Ш/В) и
  попаданию тиража в диапазон `qty_min…qty_max`.
- Если точного нет — берётся **ближайший размер** того же FEFCO (по сумме |ΔД|+|ΔШ|+|ΔВ|),
  и блок помечается «≈ похожий размер».
- Если совпадений нет вообще — блок скрывается.
- Показываются 1–3 конкурента с ценой без НДС и % отклонения нашей цены: **зелёный** — наша
  цена ниже (выгодно), **красный** — выше. Также выводится дата «Обновлено».
- Любые ошибки (нет файла, битый JSON) обрабатываются тихо — блок скрывается, основной
  расчёт не ломается.

**Как отредактировать цены (без изменения кода):**

1. Откройте файл `competitors.json` в репозитории на GitHub.
2. Нажмите значок карандаша (Edit this file).
3. Измените/добавьте записи (формат — см. ниже; пример есть в самом файле в поле `_example`).
4. Внизу нажмите **Commit changes**.
5. Изменения появятся на сайте через ~10–60 секунд (кэш CDN GitHub Pages может задержать
   обновление на несколько минут — при необходимости обновите страницу с очисткой кэша).

**Формат записи:**

```json
{
  "updated": "2026-07-07",
  "items": [
    {
      "fefco": "0201",
      "length": 400, "width": 300, "height": 250,
      "qty_min": 100, "qty_max": 299,
      "competitors": [
        { "name": "Конкурент А", "price_no_vat": 1.85, "currency": "BYN" },
        { "name": "Конкурент Б", "price_no_vat": 1.92, "currency": "BYN" }
      ]
    }
  ]
}
```

- `length`/`width`/`height` — габариты коробки в мм (Д/Ш/В), должны совпадать с вводом.
- `qty_min`/`qty_max` — диапазон тиража; `qty_max: null` означает «и выше» (без верхней границы).
- `price_no_vat` — цена конкурента за 1 шт **без НДС**.
- Поле `updated` — дата последнего обновления, показывается в блоке.

## Настройки

Откройте вкладку «Настройки» в приложении:

- отредактируйте коэффициенты наценки по градациям — **отдельно** для набора А
  (четырёхклапанные) и набора Б (самосборные), оба редактируются независимо;
- настройте до трёх правил надбавки по площади заготовки;
- добавьте пользовательские типы развёртки (переменные: A — длина, B — ширина,
  H — высота, C — клеевой клапан, D — припуск);
- укажите цену картона за м² (BYN) и плотность гофрокартона (г/м²);
- при необходимости измените ставку НДС, статьи накладных расходов и ограничения размеров;
- впишите название компании и реквизиты — они попадут в шапку PDF-бланка;
- загрузите логотип (PNG/JPG) — он появится только в PDF коммерческого предложения.

Настройки сохраняются в браузере. Кнопка «Сбросить к стандартным» возвращает исходные значения.

Значения по умолчанию: ООО БОКСМАРТ, г. Минск, тел. +375 29 616-81-69, УНП 193915237.

## Технологии

- Чистый HTML/CSS/JavaScript, без фреймворков и сборщика на клиенте.
- [jsPDF](https://github.com/parallax/jsPDF) + jsPDF-AutoTable — экспорт PDF.
- [ExcelJS](https://github.com/exceljs/exceljs) — экспорт XLSX.
- Шрифт PT Sans (встроен как base64) — корректная кириллица в PDF.
